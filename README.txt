# 🚀 JavaScript Crash Course

Welcome to the JavaScript Crash Course! This guide summarizes everything we’ve learned so far, from fundamentals to more advanced problem-solving.

---

## ✅ What We Covered

### 🔹 Console Output
- `console.log()` — Print output
- `console.error()` — Show errors
- `console.warn()` — Show warnings
- `alert()` — Pop-up messages

---

### 🔹 Variables
- `var`, `let`, `const`
- Reassignment, immutability, scope

---

### 🔹 Data Types
- `String`, `Number`, `Boolean`, `null`, `undefined`
- Use `typeof` to check types

---

### 🔹 Strings
- Concatenation and template literals
- String methods: `.length`, `.toUpperCase()`, `.substring()`, `.split()`, `.replace()`, `.replaceAll()`

---

### 🔹 Arrays
- Creating arrays and accessing elements
- Methods: `.push()`, `.pop()`, `.unshift()`, `.reverse()`, `.sort()`, `.indexOf()`
- Use `Array.isArray()` to verify array type

---

### 🔹 Objects
- Key-value pairs
- Nested objects and arrays
- Dot and bracket notation
- Add properties dynamically

---

### 🔹 Dates
- `new Date()`
- Format using `toLocaleDateString()`, `getFullYear()`, etc.

---

### 🔹 Loops
- `for`, `while`, `for...of`, `for...in`
- Used for both arrays and objects

---

### 🔹 High Order Array Methods
- `.forEach()` — Iterate without returning
- `.map()` — Return a new transformed array
- `.filter()` — Return filtered elements based on condition

---

### 🔹 Conditionals
- `if`, `else if`, `else`
- `switch` statements
- Ternary operator `condition ? true : false`

---

### 🔹 Functions
- Function declarations and default parameters
- Arrow functions `()=>`

---

### 🔹 Object-Oriented Programming (OOP)
- Constructor functions with prototypes
- ES6 `class` syntax with methods
- Encapsulation with private fields (`#`)
- Inheritance, polymorphism, method overriding

---

## 🧠 Coding Exercises We Solved

- ✅ Palindrome checker
- ✅ Anagram checker
- ✅ Longest word finder
- ✅ Binary search algorithm
- ✅ Custom `map()` + `filter()` challenge
- ✅ Problem-solving with `forEach`, `map`, `filter`
